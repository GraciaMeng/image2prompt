# 商店截图与宣传图拍摄清单

用途：为 `Image to Prompt` 准备 Chrome Web Store 提交所需的截图和宣传素材。本文聚焦“拍什么”和“文案怎么配”，不替代最终尺寸规范校验；提交前请以 Chrome Web Store 后台当期要求为准。

## 素材准备目标

建议至少准备以下几类内容：

- 扩展图标
- 商店截图
- 小宣传图 / 大宣传图（如果当前提交流程或后续推广需要）
- 可选的社媒或 README 展示图

## 截图拍摄建议

### 截图 1：设置页总览

- 目标：让用户一眼看懂“这是一个需要自配模型接口的工具”
- 建议画面：
  设置页展示 `API Key`、`Base URL`、`Model` 与历史记录两大区块
- 画面文案建议：
  `连接你自己的多模态模型接口`
- 拍摄注意：
  使用测试 Key 或打码；不要暴露真实凭据

### 截图 2：网页图片右键入口

- 目标：突出核心触发方式简单直接
- 建议画面：
  在一个干净的图片网页上，右键图片并展示“分析图片并提取 Prompt”菜单项
- 画面文案建议：
  `右键网页图片即可开始分析`
- 拍摄注意：
  优先选版权可控或自制测试图片页面

可选补拍：

- 如果要强调当前版本已有双入口能力，可再补一张“图片侧悬浮入口”截图，与右键入口分开展示。

### 截图 3：流式分析过程

- 目标：体现“不是跳转到别处”，而是在原页面直接工作
- 建议画面：
  网页右侧或图片旁的悬浮卡片处于“正在生成提示词...”状态
- 画面文案建议：
  `在当前页面直接查看流式结果`
- 拍摄注意：
  尽量选中文或双语页面，减少视觉噪音

### 截图 4：分析完成结果

- 目标：展示最终交付物长什么样
- 建议画面：
  已生成完整 prompt 的悬浮结果卡片，包含复制按钮或重试入口
- 画面文案建议：
  `快速得到可复用的 Prompt 草稿`
- 拍摄注意：
  结果文本提前准备一份质量稳定的演示样本，避免现场生成不稳定

### 截图 5：本地记录

- 目标：补足“不是一次性工具”，而是能回看真实历史
- 建议画面：
  设置页中的真实分析记录区域
- 画面文案建议：
  `回看最近分析结果`
- 拍摄注意：
  若当前界面信息密度较高，建议裁切到最能说明问题的区域；建议先完成一次真实分析，再拍摄真实记录而不是示例卡片

## 宣传图方向建议

### 方向 A：工具型主视觉

- 标题候选：
  `Turn Images Into Reusable Prompts`
- 副标题候选：
  `Analyze web images with your own multimodal model`
- 视觉元素建议：
  浏览器窗口、图片卡片、Prompt 文本流、右键触发符号

### 方向 B：中文商店页主视觉

- 标题候选：
  `右键图片，提取 Prompt`
- 副标题候选：
  `使用你自己的模型接口，在网页中直接完成分析`
- 视觉元素建议：
  网页图片、结果悬浮层、设置页字段、图标组合；如需突出双入口，可加入右键菜单与悬浮按钮

## 拍摄规范建议

- 使用干净的测试网页和可公开展示的图片素材
- 所有 `API Key`、账号名、邮箱、接口域名中的敏感部分都要打码
- 统一浏览器缩放比例、窗口尺寸和语言，避免一组截图风格不一致
- 结果文本尽量提前固定，避免因为模型随机性导致截图风格跳动
- 如需体现国际化，可以额外拍一组英文界面或英文商店素材

## 提交前人工确认项

- Chrome Web Store 当前要求的截图尺寸、数量上限和宣传图规格
- 最终使用中文还是英文作为主商店语言
- 是否需要为官网、README、社媒额外导出一套横版物料
- 截图中展示的网站、图片和文本是否具备可公开使用权限
